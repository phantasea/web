[Solved] Font is not loaded in .Xresources

I am trying to display Chinese characters in uxterm (they are currently blank). Here is my .Xresources:

*faceName: DejaVu Sans Mono
*faceNameDoublesize: WenQuanYi Bitmap Song

The output of lsof -p <xterm pid> | grep fonts contains

/usr/share/fonts/TTF/DejaVuSansMono-Oblique.ttf
/usr/share/fonts/TTF/DejaVuSansMono-Bold.ttf
/usr/share/fonts/TTF/DejaVuSansMono.ttf

This means that DejaVu is correctly loaded. If I set faceNameDoublesize to something like Liberation Mono, both DejaVu and Liberation are loaded (as seen in lsof output). So, I think my issue is with the font WenQuanYi Bitmap Song. I have read this section of the wiki.

I have installed ttf-dejavu, ttf-liberation, wqy-bitmapfont.

Did I install the right package? How can I make it load correctly?

Re: [Solved] Font is not loaded in .Xresources

With the command fc-list, I could find that the configuration should be

*faceNameDoublesize: WenQuanYi WenQuanYi Bitmap Song

Chinese characters now display correctly.

I have also updated the wiki.

Re: [Solved] Font is not loaded in .Xresources

Looks like the wqy font name is "WenQuanYi WenQuanYi Bitmap Song"  while the config you give just has "WenQuanYi Bitmap Song" (i.e., missing the first "WenQuanYi"). It's my understanding that the names have to match exactly, so I would try that first. If that turns out to be the issue, going forward, you can use fc-list(1) to see what name to use.
