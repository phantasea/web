Vundle插件管理

2012/03/17 maucat

Vundle是VIM里用来管理插件的一个强力插件。Vundle是VIM Bundle的简写。 vundle的配置都写在.vimrc文件里，从此可以一个文件走天下了。

Vundle利用版本控制工具Git直接从插件开发者的托管服务器上下载安装插件，并且只需一个命令，你就能更新你所有的插件。且VIM的官方插件在Github上都有一个副本 https://github.com/vim-scripts/

安装Vundle的前你应该安装Git版本控制工具，这是必须的。 在ArchLinux下

$ pacman -S git

安装Vundle:

$ git clone https://github.com/gmarik/vundle.git ~/.vim/bundle/vundle
$ git clone https://github.com/gmarik/vundle.git ~/.vim/bundle/vundle

简单的配置下.vimrc文件你就能用Vundle管理插件了，下面这些设置是必有的。

set nocompatible
filetype off
set rtp += ~/.vim/bundle/vundle/
call vundle#rc()

然后用命令Bundle设置你要用的插件(这些命令都写在.vimrc文件里），如： 用Vundle来管理本身

Bundle 'gmarik/vundle'

gmarik/vundle 其实是 https://github.com/gmarik/vundle.git的简写，这种简写只对于托管在github上的代码起作用。

例如一个VIM书签插件的仓库是https://github.com/dterei/VimBookmarking.git 则可在.vimrc文件里配置为

Bundle 'dterei/VimBookmarking'

如果你要安装的插件是在官方的插件副本库里，则可以写得更简单。

如：https://github.com/vim-scripts/The-NERD-tree.git可写为

Bundle 'The-NERD-tree'

如果插件没有托管在Github上，那你可就要写全地址了 如：

Bundle 'git://git.wincent.com/command-t.git'
Bundle 'https://example.com/aut/sth.git'

好了。到此就配置完Vundle了，接下来进入Vim，运行命令：

:BundleInstall

就能安装你在.vimrc文件里指定的所有插件了。 这些插件都将默认安装到~/.vim/bundle/里，安装地址不能更改，不过这些都交由Vundle开管理，所以问题不大。 你也可以在Vim通过命令

:BundleSearch

来实时搜索和安装你所要的插件，此时插件是从官方库里来安装的，插件安装好后会立即启用，但是如果你想每次都自动启用的话，你就得用Bundle命令在.vimrc文件里写上这个插件了。

接下来说下卸载，如果你不想用某个插件了，只需要在.vimrc 文件里把关于那个插件的Bundle删除，然后在Vim里运行：

:BundleClean

根据提示就能卸载了

Vundle管理插件是根据设置VIM的runtimepath(可参看:h rtp)来工作的，每增加一个插件，Vundle就在runtimepath里加上那个插件的路径，以此来搜索到插件。

VIM本身就缺少一个强大的插件管理机制，出现个Vundle来填补空缺真不错。但Vundle不满意的是，他会附带把.git，也就是插件仓库也给下到目录里，这相当于我有了一个插件的所有版本了，呵呵，不过这在网络比较慢的时候，会明显感到痛苦的。 嗯。顺带把Vundle的帮助文档也翻译了下：

https://github.com/maucat/vundle.doc

enjoy.
