解牛利刃rxvt-unicode, GNU screen

Dylan.Wen posted @ 2011年3月03日 13:15 in Unix/Linux with tags rxvt-unicode GNU screen , 954 阅读

    kde 环境下的 kconsole 十分之美观，gnome-terminal 也相当的好用，但是个头大的东西都有个不足之处——过于笨重。而 rxvt 是个轻量快速，节省内存的终端，然而不支持中文。rxvt-unicode 则是它的 unicode 版，对中文支持良好，rxvt-unicode被收录在 debian/ubuntu 的源中，安装只需在管理员权限下输入命令：
        # aptitude install rxvt-unicode
    在 debian 6 wheezy中安装好后，gnome菜单中没有它，需要手动创建快捷方式。
    默认安装好后的 rxvt-unicode 相当原始，需要配置一番才方便使用。它的配置文件为 ~/.Xresource 或 ~/.Xdefaults，关于它的配置网上已有很多讲解文章。这里简单备份下我自己的配置:

    简单解释下，rxvt-unicode 接受以 Rxvt 或 URxvt 开头的属性定义，其属性一般以驼峰风格命名。
    geometry 定义了窗口的大小，即 colum x row ,也可以指明窗口左上角在屏幕的初始位置，如
        Rxvt.geometry:        80x24+100+100
    visualBell 与 bash, cygwin 的闪屏效果一致(还有下面介绍的 screen 也支持此功能)。
    termName 表示终端名字，取 rxvt 使之与非 unicode 版本的一致(有时某些环境变量如 PS1 的设置即与此名字相关，如下面介绍的 screen)。进入 rxvt-unicode,环境变量 $TERM 即为所设置的 rxvt。
    配色方面，除了一般的背景色，前景色，光标颜色之外，rxvt-unicode 还可以特别地设置粗体，斜体，超链接，下划线等颜色，所以在 rxvt-unicode 里，你可以有一个彩色的 man 手册页(很受欢迎的一个特性)。
    滚动条方面，设置了放在右侧，当有输出或按下键盘时，即自动滚动到最下端(与标准的终端如 gnome-terminal 行动一致)。不过不显示滚动条。即使不显示，按键后自动滚动到最下端的行为还是有效的。设置了鼠标闪烁。
    编码与 locale 设置为 UTF-8。将 inheritPixmap 打开，与 transparency 结合可以让背景透明，并设置其透明度。也有其它选项，可以将某张图片应用为背景。输出缓冲为 1000 行。打开 perl 扩展，使得在单击显示的超链接可以直接打开浏览器浏览。rxvt-unicode 也直接支持多标签，只要打开相应的 perl 扩展即可。不过我试了一下，鼠标反应不好，所以没有打开。我还是习惯搭配 screen 来模拟多标签。
    字体方面，是 rxvt-unicode 多为人诟病的一个方面。首先，加入 X 字体后使得加载速度变慢(一般用户感觉不到，即使将 preeditType 设为 Root)，有人说小字体用点阵，大字体用 X 字体比较美观，不过外观这东西见人见智。我统一用 X 字体，这里使用的就是传说中的 Vera Sans 圆体。
    更改配置之后，运行命令行
        $ xrdb ./.Xresources
    使之生效。

    GNU screen 是命令行下的窗口管理器，它主要功能包括命令行会话管理，多标签环境模拟，它在 debian/ubuntu 下的安装也很方便：
        # aptitude install screen

# 关闭启动画面
startup_message off     # default: on
 
# Automatically detach on hangup, or the screen will keep eating
# your memory after the terminal emulator has been terminated.
autodetach on      # default: on
 
# 关闭vbell
vbell off
 
# 指定 shell
shell bash
 
# 默认开始路径为~
# chdir
 
# 定义回滚行数
defscrollback 1000
 
# 指定编码
defencoding utf-8
encoding utf-8 utf-8
 
# 定制 caption
# backtick 47 0 0 my_name.sh
# caption always "%?%F%{-b 4w}%:%{-b bb}%? %H | %l | %m-%d %c |%?%F%{-b 4w}%?%L=%-Lw%45>%{-b w4}%n%f* %t%{-}%+Lw%-0<"
# caption always "%{=b}%{b}%{uB C}%47`%{-}%{B C}:%{-}  %-Lw%{u .my}%50>%n%f* %t%{-}%+Lw%< %=%{b km}%m-%d %c %D"
 
caption always "%{=b}%{b}%{C}%H%{-}  %+010=%-w%8L>%{uB .my}*%n %t%{-}%+w%-014<  %-012=%{b m}%m-%d %c "
 
# hardstatus alwayslastline "%?%{yk}%-Lw%?%{wb}%n*%f %t%?(%u)%?%?%{yk}%+Lw%?"
 
# 定制 title bar
term rxvt
 
# termcapinfo xterm|rxvt ti@:te@
# termcapinfo xterm 'hs:ts=\E]2;:fs=\007:ds=\E]2;screen\007'
termcapinfo rxvt 'hs:ts=\E]2;:fs=\007:ds=\E]2;screen\007'
 
########## 键绑定 ##########
 
# 将快捷键默认 C-a 设为 C-l
escape ^L^L
 
# 定制 region 函数键与Emacs一致
bind 2 split
bind o focus
bind 1 only
bind 0 remove
 
# bind F9 to create a new screen
bindkey -k k9 screen
# bind F10 to kill current screen window
bindkey -k k10 kill
 
# bind F5 to detach screen session
bindkey -k k5 detach
# bind F11 to "move to previous window"
bindkey -k F1 prev
# bind F12 to "move to next window"
bindkey -k F2 next
 
 
# .screenrc ends here

    caption是按照 80 个字符宽来设定的。region 函数键设定与 Emacs 一致（主要是为了方便记忆，我没有在 screen 中使用 Emacs 的习惯），可惜只能横向分割窗口，不能纵向分割。快捷键对照如下：
    F9 -- 新建 screen 会话
    F10 -- 关闭当前会话
    F5 -- 挂起当前会话
    F11 -- 移动到上一个会话
    F12 -- 移动到下一个会话
