GIT笔记

2013/11/03 maucat

设置

--system 为全局设置 设置的文件为/etc/gitconfig
--global 为当前用户设置 设置的文件为~/.gitconfig
另一个配置文件为工作目录下的 .git/config

上面三个配置文件级别从低到高，高级别的配置会覆盖上级别的相同配置

姓名与邮件

$ git config --global user.name "Maucat"
$ git config --global user.email zjc009@gmail.com

别名

为git commit创建别名git ci:

  • 全局： $ git config --system alias.ci commit

  • 本用户： $ git config --global alias.ci commit

开启颜色

$ git config --global color.ui true

默认编辑器

$ git config --global core.editor vim

差异分析工具

$ git config --global merge.tool vimdiff

查看配置信息

$ git config --list  //查看所有的配置信息：
$ git config user.name  //查看某个环境变量的设定

建立仓库

初使化

$ git init  //初始化

在一个目录下执行此命令后，该目录被初始化为新仓库

追踪文件

$ git add *.c

用git add命令告诉Git开始对这些文件进行跟踪

提交到版本库

$ git commit -m "About this commit"

$ git commit -a "跳过使用暂存区域，即跳过 git add "

打开编辑器编辑提交说明：

$ git commit

编辑完成后退出编辑器即为提交，提交说明不能只包含注释，否则不能提交成功

忽略某些文件

在工作目录下创建一个.gitignore文件来说明忽略哪些文件，可使用shell所使用的简化了的正则表达式。

  • #号开头的行视为注释
  • 简化的正则表达式
    • 星号*匹配零个或多个任意字符
    • [abc]匹配a、b、c其中的一个，[0-9]匹配数字1到9
    • 问号?匹配一个任意字符
  • 匹配模式后跟/表示要忽略的是目录
  • 前面加上!表示取反

例子：

# 此为注释 – 将被 Git 忽略
# 忽略所有 .a 结尾的文件
*.a
# 但 lib.a 除外
!lib.a
# 仅仅忽略项目根目录下的 TODO 文件，不包括 subdir/TODO
/TODO
# 忽略 build/ 目录下的所有文件
build/
# 会忽略 doc/notes.txt 但不包括 doc/server/arch.txt
doc/*.txt

取得仓库

取得仓库也就是克隆一个仓库，命令格式为git clone [url]

$ git clone git://github.com/schacon/grit.git

该命令在当前目录下创建一个名为 grit 的目录，里面包含版本库.git与及最新版本的文件拷贝。如果在clone的时候更改目录名，这样做：

$ git clone git://github.com/schacon/grit.git mygrit

当前状态与比较

状态查看

$ git status

改动比较

  • 工作区与暂存区：git diff
  • 工作区与当前分支：git diff HEAD
  • 暂存区与版本库：git diff --cached或git diff --staged

删除 不再追踪

删除工作目录下文件并且不再追踪：

$ git rm file
$ git rm -f file  # 如果将删除的文件已经被暂存，则用此命令

不再追踪，但不删除工作目录下文件：

$ git rm --cached file

shell匹配模式不会递归，加上\来递归匹配（也即用git的匹配模式)，删除当前目录下及其子目录中所有.log结尾的文件。

$ git rm \*.log

移动或者更名

$ git mv file_from file_to

查看提交历史

$ git log

一些可用的参数为：

-p 显示每次提交差异
-2 显示最近2次提交，可以是-3等，嘿嘿
--stat 简要显示增改行数统计

基本操作

撤消操作

如果提交后，暂存区为空，则运行

$ git commit --amend

相当于重写上一次的提交说明

如果提交后，暂存区又增加了新的快照，则运行此命令相当于本次提交与上一次的提交视为同一个提交，提交说明保存第二次提交时写的

远程仓库

查看项目远程仓库

$ git remote -v

添加远程仓库

$ git remote add [shortname] [url]

shortname为url的缩写，可用自己容易记住的字符串代替

从远程仓库抓取数据

$ git fetch [remote-name]

抓取远程机器上的所有更新，但是并不自动合并到当前工作分支，如果要自动合并，使用下面命令

$ git pull [remote-name]

如果单单运行命令，则默认的远程机器为origin，分支为master

推送数据到远程仓库

$ git push [remote-name] [branch-name]

如果单单运行命令，则默认的远程机器为origin，分支为master

查看远程仓库信息

$ git remote show [remote-name]

远程仓库删除与重命名

$ git remote rename old new

修改远程仓库在本地的简称old修改为new

打标签 tag

主要用途：区别软件版本

列出已有标签

$ git tag

创建一个含附注的标签

$ git tag -a v1.4 -m 'my version 4.1'

查看特定的标签：

$ git show v1.4

创建一个轻量级标签

$ git tag v1.4

git push1默认不会把标签传送到远端服务器上，如果要传送特定标签，可用下面命令：

$ git push origin [tagname]

如果要传送所有的新增标签：

$ git push origin --tags

分支

创建分支：

$ git branch [branch-name]

注意：创建分支后并不会切换到该分支上

切换分支：

$ git checkout [branch-name]

新建并切换到该分支：

$ git checkout -b [branch-name]

合并分支（如当前在master上，要合并hotfix）：

$ git merge hotfix

删除一个分支（-D为强制删除）：

$ git branch -d [branch-name]

查看分支清单：

$ git branch

不加任何参数，则列出所有分支的清单，当前分支前有*字符

查看已被并入当前分支的分支：

$ git branch --merged

查看未被并入当前分支的分支：

$ git branch --no-merged

推送本地分支

$ git push (远程仓库名) (分支名)

把本地分支推送到远程命名不同的分支：

$ git push （远程仓库名） branchname:remote_branchname

跟踪远程分支：

$ git checkout -b [分支名] [远程名]/[分支名]

删除远程分支：

$ git push [远程名] :[分支名]
